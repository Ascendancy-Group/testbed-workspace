- Testing direct SSH command confirmed MCP server functional
- OpenClaw gateway was spawning correct process but wrong palace path

**Solution:**
- Updated openclaw.json to pass `--palace /opt/mempalace` as CLI argument
- Ran `mempalace --palace /opt/mempalace mine /opt/mempalace` to initialize
- Verified with `mempalace__mempalace_status` returning correct path
